# Acme Corp Engineering Standards

Generated by Temper
Based on scan of acme-payments-service

## Mandatory Rules (BLOCK if violated)
- Never expose database entities in API responses — use DTOs
- All database access must go through repository layer
- PII must be encrypted at rest
- No secrets in source code — use environment variables or Vault
- All user input must be validated before processing

## Quality Rules (WARN if violated)
- Methods should not exceed 30 lines
- Classes should not exceed 300 lines
- Constructor injection only (no field injection)
- Structured logging with correlationId in every log statement
- Test coverage must be >= 85% for new code

## Conventions (SUGGEST improvements)
- Error codes follow ACM-XXXX format
- Branch names: feature/ACM-{ticket}-{description}
- PR descriptions must reference the Jira ticket
- Commit messages follow Conventional Commits

## Architectural Constraints (BLOCK if violated)
- All external API calls go through the gateway service
- No circular dependencies between domain modules
- Event consumers must be idempotent
- Database migrations must be backward-compatible
