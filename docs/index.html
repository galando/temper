Why Temper

Workflow

The User Flow

Four steps from idea to production-ready code. Temper automates quality — you focus on shipping.

01 You type

Plan /temper:plan

You describe what you want to build. Temper automatically analyzes the blast radius — mapping affected files, dependencies, and risk areas.

✓ 12 files affected · ✓ Dependencies mapped · ! 2 risk areas
02 Automated

Build /temper:build

Temper implements automatically with TDD gates enforced at every step. Tests written first, code follows. Cannot proceed until tests pass.

✓ 18 tests pass · ✓ 86% coverage · ✓ Gates satisfied
03 Automated

Review /temper:review

Temper reviews automatically against enabled packs. Findings scored by confidence — high-confidence issues require your attention.

✓ 94% confidence · 3 findings · 1 auto-fixed · 1 needs review
04 Automated

Track & Learn /temper:status

Temper learns automatically over time. Tracks patterns, coverage trends, and suggests new rules based on your codebase.

Quality Packs

Built-in Packs

Rules that enforce best practices. Each pack has BLOCK, WARN, and SUGGEST severity levels.

💎

Quality BLOCK

  • Methods max 30 lines
  • No duplicate code (DRY)
  • Max complexity: 10
🧪

TDD WARN

  • Write test first (RED)
  • Minimal implementation
  • Coverage: 80%
🔒

Security BLOCK

  • No SQL injection
  • No XSS vulnerabilities
  • No secrets in code
🌿

Git SUGGEST

  • Conventional commits
  • Branch conventions
  • Atomic commits

🎨 Create Your Own Packs

Add a rules.md file and Temper will automatically discover it. Perfect for company-specific standards.

.claude/packs/my-company/rules.md # My Company Standards ## BLOCK - All API responses use DTOs - No raw SQL queries ## WARN - Constructor injection only - Max method length: 20 lines

⚡ Or create a pack interactively

STEP 1
Run the command

Type /temper:standards in any project

STEP 2
Answer questions

Temper scans your codebase and asks about patterns it finds

STEP 3
Get your pack

Temper generates a custom rules.md based on your answers
